## 目标
- 提供一份可直接复制使用的提示词，将 PDF 试卷内容转成 Markdown，完全符合站点解析器（assets/js/exams.js）支持的格式。

## 解析兼容要点
- 顶部必须有一级标题：`# <试卷标题>`（如 `# 2022年三套卷2`）。
- 每题用三级标题：`### <题目标题>` 开始；随后允许元信息行：`tags:`、`difficulty:`、`knowledge:`。
- 正文区块关键锚点：`题目:`、（可选）`选项:`、`答案:`、`解析:`。
- 选项识别规则：选项行必须以大写字母加标点开头，如 `A.`、`B.`、`C.`、`D.`、`E.`（或 `(A)`/`（A）`）。
- 数学：行内 `$...$`；块级 `$$...$$`。
- 可选的分学科/分题型分节：`### 数据结构`，`一、选择题` 等，解析器可识别但非必需。

## 输出格式规范（推荐统一使用“结构化块写法”）
- 每题模板如下（按题型调整是否包含“选项:”区块）：

```
# <试卷标题>

### <题目标题>
tags: ["数据结构", "<主题标签>"]
difficulty: <简单|中等|较难>
knowledge: <对应知识点>
题目:
<题干文本，保留必要换行>

选项:
A. <选项A>
B. <选项B>
C. <选项C>
D. <选项D>
E. <选项E>

答案: <单选写字母；多选写AB或A,B；判断写对/错或√/×；填空/解答写文本答案>
解析:
<关键理由、步骤、公式；可含$$块级公式$$或$行内$公式>
```

- 判断题/填空题/解答题：省略“选项:”区块，其他保持一致。
- 多题目依次重复上述块，每题之间保留一个空行。

## 注意与清洗
- 选项行不要加列表符号（如 `- A.`）；必须是行首 `A.` `B.` 等。
- 题干、解析中不要添加 Markdown 引用符号（`>`）或多余粗体装饰；保持纯文本与公式。
- 清除 PDF 页眉页脚、题号混排、分页残留；最多允许两连空行。
- 中文内容保持原文，全角标点可保留；去除不可见字符与乱码。

## 提示词（直接复制使用）
```
你是专业排版助手。请将我提供的“PDF试卷文本”转换为符合以下规则的 Markdown：

[输出目标]
- 仅输出 Markdown，不要解释或附加额外文本。
- 必须使站点解析器可正确识别并渲染试卷与题目。

[全局结构]
1) 顶部：一级标题 `# <试卷标题>`（从原文标题提取）。
2) 题目：使用“结构化块写法”，每题以 `### <题目标题>` 开始，然后按顺序给出以下区块：
   - `tags: ["数据结构", "<主题标签>"]`（JSON数组，至少包含学科标签，可根据题干自动补充如“红黑树”“B树”“KMP”等）。
   - `difficulty: <简单|中等|较难>`（依据题目难度判断）。
   - `knowledge: <对应知识点>`（一句话概括关键知识点）。
   - `题目:` 后跟题干正文，保留必要换行。
   - 若为选择题：加入 `选项:`，每个选项占一行，严格以 `A.`、`B.`、`C.`、`D.`、`E.` 开头；不要使用列表或缩进符号。
   - `答案:` 一行给出答案：
       - 单选：字母（如 `C`）
       - 多选：`AB` 或 `A,B`（任选其一）
       - 判断：`对`/`错` 或 `√`/`×`
       - 填空/解答：直接写文本答案
   - `解析:` 多行，给出关键推理、步骤、结论，可含公式与简表。

[数学与符号]
- 行内公式使用 `$...$`；块级公式使用 `$$...$$`，不要在公式外添加额外的 Markdown 装饰。
- 规范化被 PDF 破坏的分数、上下标、希腊字母等；必要时将复杂公式改为 LaTeX 表达。

[文本清洗]
- 删除页眉/页脚、分页号、无意义的断行与多余空格；最多保留两连空行。
- 移除所有图片/矢量残留，将表格转为普通文本或简化为要点列表（但选项区必须保持 `A.` 行格式）。

[一致性检查]
- 每题必须包含 `答案:`；选择题必须提供 `选项:` 且选项字母为大写。
- 不要在 `答案:` 与 `解析:` 之间加入其他标头；题目之间用一个空行分隔。
- 确保所有中文/英文字符正确解码，无乱码。

现在开始按以上规范输出 Markdown。
```

## 示例片段
- 单选题：
```
### 红黑树的性质
tags: ["数据结构", "红黑树"]
difficulty: 中等
knowledge: 黑高一致与红节点限制
题目:
下列关于红黑树的叙述，正确的是：

选项:
A. 红色节点的两个子节点必须都是红色
B. 任一路径的黑色节点数相同
C. 根节点必须是红色
D. 叶子必须是红色

答案: B
解析:
红黑树要求各根到叶路径黑高一致；红色节点的子节点必须是黑色等。
```

- 判断题：
```
### Fibonacci 查找比较次数
tags: ["数据结构", "查找"]
difficulty: 简单
knowledge: 期望与最坏比较次数
题目:
有序向量做 Fibonacci 查找，最坏情况下成功与失败的比较次数相等。

答案: √
解析:
最末次比较的分支分别对应成功与失败，比较次数对齐。
```
